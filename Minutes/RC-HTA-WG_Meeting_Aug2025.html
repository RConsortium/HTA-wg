Working Stream 2: Deep Dive

Building a Trusted R Package Ecosystem for HTA

WS2 Mission & Objectives

As stated in GitHub Discussion #5:

Mission: Create a curated, validated inventory of R packages for HTA analytics

Objectives: - Establish transparent assessment criteria - Build searchable package repository - Provide validation documentation - Enable community contributions

Why This Matters:

  • HTA-Body confidence in R-based submissions
  • Reduced duplication of validation efforts
  • Accelerated adoption through shared resources
  • Quality assurance through community review

Assessment Framework: Three Dimensions

Based on cross-industry input (OpenStatsWare, R Validation Hub):

1. Software Engineering (SWE) Aspects

2. Scientific Validity Aspects

3. Adoption Readiness Aspects

Each dimension includes both automated and manual assessment components

SWE Aspects: Technical Quality

ü§ñ Automated Assessment

  • Code coverage metrics (>80% target)
  • Documentation completeness (roxygen2, vignettes)
  • Dependencies analysis and version tracking
  • CI/CD pipeline status
  • CRAN checks passing status

üë• Manual Review Required

  • Code architecture quality and design patterns
  • Security vulnerability assessment
  • Performance benchmarking for large datasets
  • Error handling robustness

Scientific Aspects: Method Validity

ü§ñ Automated Assessment

  • Test cases against known results
  • Reference implementations comparison
  • Numerical stability checks

üë• Manual Review Required

  • Statistical methodology correctness
  • Alignment with HTA guidelines (NICE, IQWiG, HAS)
  • Clinical relevance of outputs
  • Peer review by domain experts (minimum 2 reviewers)

Adoption Aspects: Production Readiness

ü§ñ Automated Assessment

  • Download statistics from CRAN/GitHub
  • Issue tracker activity and response times
  • Update frequency and maintenance status
  • Reverse dependencies count

üë• Manual Review Required

  • Use cases in actual HTA submissions
  • User community feedback and support
  • Training materials availability
  • Integration with existing workflows
